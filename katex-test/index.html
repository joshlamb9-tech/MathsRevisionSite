KaTeX Rendering Test

This page verifies all KaTeX equation types render correctly. If any equation shows as raw text (e.g. $A = \pi r^2$), something is broken.

Test 1: Inline equations

The area of a circle is $A = \pi r^2$.

Pythagoras: $a^2 + b^2 = c^2$.

Expected: equations render as formatted maths inline with text.

Test 2: Display (block) equations

The quadratic formula:

$$x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$$

Expected: centred on its own line, larger than inline.

Test 3: Prime notation (Jekyll pitfall)

The derivative of $f(x)$ is written as $f'(x)$.

On GitHub Pages without .nojekyll, the apostrophe in $f'(x)$ becomes a curly quote and KaTeX throws a parse error. If this renders, .nojekyll is working.

Also: $g''(x)$ (double prime) and $h'''(x)$ (triple prime).

Test 4: Long equation (mobile overflow test)

This equation should scroll horizontally on a narrow screen, not get cut off:

$$\frac{d}{dx}\left[\int_{a}^{x} f(t)\,dt\right] = f(x) \quad \text{(Fundamental Theorem of Calculus)}$$

On mobile: scroll right inside the equation box. The page itself should not scroll horizontally.

Test 5: Nested fractions

$$\frac{1}{1 + \frac{1}{1 + \frac{1}{x}}}$$

Expected: three levels of fractions render correctly without overlap.

Test 6: Mixed inline and display

When $n$ is a positive integer, the sum of the first $n$ natural numbers is:

$$S_n = \frac{n(n+1)}{2}$$

For example, when $n = 10$, we get $S_{10} = \frac{10 \times 11}{2} = 55$.
